### 项目经历：智能简历问答 AI Agent (Resume-Link Agent)

* **项目名称与角色**：智能简历问答 AI Agent / 核心开发者（架构设计与算法实现）
* **核心技术栈**：Spring Boot 3, LangChain (Java SDK), Qdrant (向量数据库), OpenAI API, Redis, ConversationSummaryBufferMemory
* **项目背景**：解决传统静态简历信息密度低、无法根据面试官关注点进行动态交互的问题，通过 AI 实现个人能力的“深层检索”与“精准匹配”。
* **架构设计（亮点）**：
* **两阶段检索架构 (RAG)**：设计了基于语义向量检索 (Dense Retrieval) 与元数据过滤 (Metadata Filtering) 的两阶段方案。通过将个人经历标签化（如 `#架构设计`、`#Java底层`），确保 Agent 在处理模糊语义和精准术语时都能准确召回。
* **动态上下文压缩**：为了在长对话中保持“记忆”且不突破 Token 限制，采用了 `ConversationSummaryBufferMemory` 策略。将早期的对话细节自动摘要，仅保留最近 3-5 轮的高保真上下文，实现了记忆深度与 API 成本的平衡。


* **挑战与痛点**：
* **检索幻觉与相关性问题**：初期测试发现，当面试官提问过于笼统时，RAG 容易召回无关的项目片段。
* **解决方案**：引入了 **Reciprocal Rank Fusion (RRF)** 融合排序，并增加了 **Re-rank (重排序)** 环节。同时，利用 Redis 缓存高频问题的回答，将端到端响应延迟 (Latency) 降低了约 40%。


* **量化结果**：
* **响应速度**：通过 Redis 缓存与流式输出 (Streaming)，首字响应时间从 1.5s 优化至 0.4s。
* **交互深度**：相比传统简历，面试官获取特定技术细节的效率提升了 3 倍以上。
* **准确率**：在 50+ 次模拟面试追问测试中，核心技术方案的召回准确率达到 95% 以上。
