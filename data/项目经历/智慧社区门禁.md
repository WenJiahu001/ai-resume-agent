### 项目经历：智慧社区 SaaS 化运营平台

* **项目名称与角色**：智慧社区 SaaS 化运营平台 / 后端架构师、技术负责人
* **核心技术栈**：Spring Cloud, Nacos, Apache Mina, RabbitMQ, Redis, Jenkins Pipeline
* **项目背景**：以社区门禁为核心切入点，通过为昆明、四川等地 20 多个社区提供免费门禁服务，构建包含广告投放、积分商城、定点洗车等业务在内的社区商业化运营生态。
* **架构设计（亮点）**：
* **分布式长连接服务**：基于 **Apache Mina** 设计并实现了高性能 TCP 长连接网关，支持单机 5,000+ 智能门禁设备的并发接入。通过自定义心跳检测与断线重连机制，确保了设备在线状态的实时性，实现毫秒级开门响应。
* **指令中转通信架构**：舍弃传统的低效 HTTP 轮询模式，搭建基于 **RabbitMQ** 的指令异步中转架构。通过 Topic 交换机实现设备指令的精准路由与广播，将端到端控制延迟压低至 50ms 以内。
* **多租户 SaaS 体系设计**：作为技术负责人，设计并实现了支持代理商、物业、商家、广告主多方参与的 SaaS 权限与数据隔离架构，支撑了积分运营、广告投放等多元化业务的快速平访。


* **挑战与痛点**：
* **海量设备状态同步压力**：在设备密集上线阶段，频繁的状态更新容易冲击数据库与 MQ 队列。
* **解决方案**：引入 Redis 缓存设备实时位图，并配合 RabbitMQ 的死信队列与延迟插件处理异常告警。建立了一套自动化监控系统，将设备异常发现时间缩短至 30s 以内。
* **运维效率低下**：针对多模块微服务部署慢的问题，落地 Jenkins Pipeline 自动化流水线，实现了零停机滚动更新（Rolling Update）。


* **量化结果**：
* **性能表现**：指令延迟较传统 HTTP 方案提升了 **20 倍**；单机支撑 **5,000+** 智能终端并发稳定运行。
* **业务规模**：成功覆盖 **20+** 大型社区，实现了从 0 到 1 的概念落地与正式商业化运营。
* **可靠性**：通过自动化监控与告警体系，系统可用性达到 **99.9%**。
